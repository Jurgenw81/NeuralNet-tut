Chapter 00

Welcome to Neural Networks

This interactive tutorial will take you from the very basics of a single neuron all the way through to convolutional neural networks (CNNs). Every concept is explained in detail and paired with live, interactive visualizations you can play with.

What You'll Learn

Neurons Activations Layers Backpropagation

Starting with the fundamental building block — a single neuron that takes inputs, applies weights, and produces an output — you'll progressively build up to full network architectures, understand how they learn through backpropagation, and see how CNNs exploit spatial structure in images.

The Big Picture

A neural network is a function approximator. Given inputs (like pixel values of an image), it produces outputs (like "this is a cat"). It learns by adjusting internal parameters (weights and biases) to minimize the difference between its predictions and the actual answers.

The learning loop is: Forward Pass (make a prediction) → Compute Loss (how wrong were we?) → Backpropagation (figure out which parameters caused the error) → Update Weights (adjust parameters to reduce error). Repeat thousands of times.

Chapter 01

The Neuron

A neuron is the fundamental computation unit. Inspired by biological neurons, it takes multiple inputs, multiplies each by a weight, sums them up, adds a bias, and passes the result through an activation function.

output = activation( w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + bias )

Step-by-Step Breakdown

1. Inputs (x₁, x₂, ...)

These are the data flowing into the neuron. In the first layer, these are your raw features (pixel values, measurements, etc.). In deeper layers, these are outputs from previous neurons.

2. Weights (w₁, w₂, ...)

Each input has an associated weight. Weights determine how important each input is. A large positive weight amplifies the input; a large negative weight inverts it. A weight near zero means the neuron mostly ignores that input. These are the parameters the network learns.

3. Weighted Sum (z)

Multiply each input by its weight and add them all up: z = Σ(wᵢ · xᵢ). This is a linear combination — essentially a dot product between the input vector and weight vector.

4. Bias (b)

A bias is added to the weighted sum: z = Σ(wᵢ · xᵢ) + b. The bias allows the neuron to shift its activation threshold. Without bias, the neuron's decision boundary must pass through the origin.

5. Activation Function

The weighted sum + bias is passed through a non-linear activation function. Without this, stacking layers would be pointless (a stack of linear functions is still linear). The activation introduces non-linearity, allowing the network to learn complex patterns.

Chapter 02

Activation Functions

Activation functions introduce non-linearity. Without them, a neural network — no matter how many layers — would collapse into a single linear transformation. The choice of activation function affects how the network learns and what problems it can solve.

Sigmoid: σ(z) = 1 / (1 + e⁻ᶻ)

Squashes output to (0, 1). Historically popular but suffers from vanishing gradients — for very large or small z, the gradient approaches zero, making learning extremely slow in deep networks. Still used in output layers for binary classification.

Tanh: tanh(z) = (eᶻ - e⁻ᶻ) / (eᶻ + e⁻ᶻ)

Squashes output to (-1, 1). Zero-centered, which helps with gradient flow compared to sigmoid. But still has vanishing gradient problems at extremes.

ReLU: max(0, z)

Returns z if positive, 0 otherwise. Simple, fast, and the default choice for hidden layers. Solves vanishing gradient for positive values. Problem: "dying ReLU" — neurons that output 0 for all inputs stop learning entirely since the gradient is always 0.

Leaky ReLU: max(0.01z, z)

Like ReLU but allows a small gradient (0.01) for negative values. This prevents the dying ReLU problem while maintaining most of ReLU's benefits.

Softmax (for output layers)

Converts a vector of raw scores into a probability distribution that sums to 1. Used in the final layer for multi-class classification. softmax(zᵢ) = eᶻⁱ / Σeᶻʲ

Chapter 03

Layers & Architecture

Neurons are organized into layers. A typical feedforward neural network has an input layer, one or more hidden layers, and an output layer. Each neuron in one layer connects to every neuron in the next — called a fully connected (or dense) layer.

Input Layer

Not really "neurons" — just placeholders that pass your raw data into the network. The size equals the number of features in your data (e.g., 784 for a 28×28 grayscale image).

Hidden Layers

Where the actual learning happens. Each layer transforms its input through weighted sums, biases, and activations. More layers = ability to learn more abstract features. The first hidden layer might detect edges; deeper layers might detect shapes, faces, or concepts.

Output Layer

Produces the final prediction. For classification: one neuron per class with softmax activation. For regression: a single neuron with linear (no) activation. For binary classification: one neuron with sigmoid activation.

Depth vs. Width

Width (neurons per layer) affects the network's capacity within each level of abstraction. Depth (number of layers) affects the network's ability to learn hierarchical features. Deep networks are generally more parameter-efficient than wide-shallow ones for complex tasks.

Chapter 04

Forward Pass

The forward pass is when data flows from input to output through the network. At each layer, we compute the weighted sum of inputs, add the bias, apply the activation function, and pass the result to the next layer.

For each layer l: z⁽ˡ⁾ = W⁽ˡ⁾ · a⁽ˡ⁻¹⁾ + b⁽ˡ⁾ then a⁽ˡ⁾ = activation(z⁽ˡ⁾)

Step-by-step for a 2-layer network

Input: x = [x₁, x₂] — your raw data.

Layer 1: Compute z₁ = W₁·x + b₁ (matrix multiplication), then a₁ = ReLU(z₁). The output a₁ is a vector of activations — one per neuron in this layer.

Layer 2 (output): Compute z₂ = W₂·a₁ + b₂, then ŷ = sigmoid(z₂) or softmax(z₂). This is your prediction.

We save all intermediate values (z and a at each layer) because backpropagation needs them to compute gradients.

Chapter 05

Loss Functions

The loss function measures how wrong the network's prediction is compared to the actual target. Training a neural network means minimizing this loss. The choice of loss function depends on the task.

Mean Squared Error (MSE) — Regression

MSE = (1/n) · Σ(yᵢ - ŷᵢ)²

Squares the difference between predicted and actual values, then averages. Penalizes large errors heavily due to squaring. Used when predicting continuous values (prices, temperatures, etc.).

Binary Cross-Entropy — Binary Classification

BCE = -(1/n) · Σ[ yᵢ·log(ŷᵢ) + (1-yᵢ)·log(1-ŷᵢ) ]

Measures the "distance" between two probability distributions: the predicted probabilities and the true labels (0 or 1). When the prediction is confident and correct, loss is low. When confident and wrong, loss shoots up dramatically — this creates a strong gradient signal for learning.

Categorical Cross-Entropy — Multi-class Classification

CCE = -Σ yᵢ · log(ŷᵢ) (summed over all classes)

Extension of binary cross-entropy for multiple classes. Used with softmax output. Only the true class contributes to the loss (since yᵢ = 0 for all incorrect classes).

Chapter 06

Backpropagation

Backpropagation is the algorithm that computes gradients of the loss with respect to every weight in the network. It uses the chain rule of calculus to propagate the error signal backwards from the output layer to the input layer.

Why Backprop?

To update weights, we need to know: "If I change this weight by a tiny amount, how much does the loss change?" This is the gradient ∂L/∂w. Computing this directly for every weight would be impossibly expensive. Backprop does it efficiently by reusing intermediate computations.

The Chain Rule

∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w

The loss depends on the activation, which depends on the weighted sum, which depends on the weight. By chaining these derivatives together, we can compute how each weight contributed to the final error.

Step-by-Step

1. Compute output error: ∂L/∂ŷ — how does the loss change with the prediction? For MSE: 2(ŷ - y). For cross-entropy with sigmoid: (ŷ - y).

2. Propagate through activation: Multiply by the derivative of the activation function. For sigmoid: σ(z)·(1-σ(z)). For ReLU: 1 if z>0, else 0.

3. Compute weight gradients: ∂L/∂W = δ · aᵀ (the error signal times the input to that layer).

4. Propagate to previous layer: δ_prev = Wᵀ · δ — send the error backward through the weights.

5. Repeat for each layer from output to input.

Chapter 07

Optimizers & Learning

Once we have gradients from backprop, we need to actually update the weights. The optimizer determines how we use those gradients. The simplest approach is gradient descent, but modern optimizers are much smarter.

Stochastic Gradient Descent (SGD)

w = w - lr · ∂L/∂w

Subtract the gradient (scaled by the learning rate) from each weight. Simple but can oscillate in ravines and get stuck in local minima. "Stochastic" because we compute gradients on mini-batches, not the full dataset — this adds noise that actually helps escape bad local minima.

SGD with Momentum

v = β·v + ∂L/∂w then w = w - lr · v

Maintains a velocity vector that accumulates past gradients. This smooths out oscillations and accelerates learning in consistent gradient directions — like a ball rolling downhill with momentum.

Adam (Adaptive Moment Estimation)

Combines momentum (first moment) with RMSProp (second moment — tracking gradient magnitude per parameter). Each weight gets its own adaptive learning rate. The current default choice for most tasks. Converges faster and requires less hyperparameter tuning than SGD.

Learning Rate

The most critical hyperparameter. Too high → training diverges (loss explodes). Too low → training is extremely slow and may get stuck. Common strategy: start with a moderate rate (0.001 for Adam) and decay it over time using a scheduler.

Chapter 08

Convolutional Neural Networks

CNNs are specialized neural networks designed for data with spatial structure — primarily images. Instead of connecting every input to every neuron (fully connected), CNNs use local connectivity and weight sharing through convolution operations.

Why Not Use Regular NNs for Images?

A 224×224 RGB image has 150,528 input values. A single fully-connected hidden layer with 1000 neurons would need 150 million weights — just for one layer. This is wasteful because images have local structure: nearby pixels are related, and a pattern (like an edge) looks the same regardless of where it appears.

Key CNN Ideas

Local Connectivity: Each neuron only looks at a small patch of the input (the receptive field), not the entire image.

Weight Sharing: The same small set of weights (a filter/kernel) is applied across the entire image. This means the network automatically detects a feature regardless of its position (translation invariance).

Hierarchy of Features: Early layers detect simple patterns (edges, colors). Middle layers combine these into textures and parts. Deep layers recognize objects and scenes.

Typical CNN Architecture

Input → [Convolution → Activation → Pooling] × N → Flatten → Fully Connected → Output

Chapter 09

The Convolution Layer

The convolution operation slides a small filter (kernel) across the input image, computing a dot product at each position. This produces a feature map — a 2D map showing where the filter detected its pattern.

output[i,j] = Σₘ Σₙ input[i+m, j+n] · kernel[m, n] + bias

The Kernel/Filter

A small matrix (typically 3×3 or 5×5) of learnable weights. Different kernels detect different features. A horizontal edge kernel might look like [[1,1,1],[0,0,0],[-1,-1,-1]]. The network learns what kernels are useful through backpropagation.

Stride

How many pixels the kernel moves at each step. Stride 1 means it moves one pixel at a time (maximum resolution). Stride 2 skips every other position, halving the output dimensions.

Padding

"Same" padding adds zeros around the border so the output has the same dimensions as the input. "Valid" padding uses no padding, so the output shrinks. Same padding preserves spatial dimensions; valid padding loses border information.

Multiple Filters

Each conv layer has multiple filters (e.g., 32 or 64), each producing its own feature map. These are stacked as channels in the output — so a layer with 32 filters turns a single-channel input into a 32-channel output.

Chapter 10

Pooling & Stride

Pooling layers reduce the spatial dimensions of feature maps, decreasing computation and making the network more robust to small translations in the input. The most common type is max pooling.

Max Pooling

Divides the feature map into non-overlapping patches (e.g., 2×2) and takes the maximum value from each patch. This halves the width and height. The intuition: if a feature was detected anywhere in the patch, we keep it — the exact position doesn't matter.

Average Pooling

Same as max pooling but takes the average instead. Provides a smoother downsampling. Sometimes used in the final layer (Global Average Pooling) to replace the fully connected layers.

Why Pooling Helps

Dimensionality reduction: Fewer parameters in subsequent layers = less computation and less overfitting.

Translation invariance: Small shifts in the input don't change the pooled output.

Larger receptive field: After pooling, each neuron in the next conv layer effectively "sees" a larger portion of the original image.

Chapter 11

Full CNN Pipeline

Let's put it all together. A complete CNN takes a raw image, passes it through alternating convolution and pooling layers to extract hierarchical features, flattens the result, and feeds it through fully connected layers for classification.

Example Architecture (LeNet-style)

Input: 28×28 grayscale image (e.g., a handwritten digit)

Conv1: 6 filters of 5×5 → 6 feature maps of 24×24. Detects simple edges and strokes.

Pool1: 2×2 max pooling → 6 maps of 12×12. Reduces spatial dimensions.

Conv2: 16 filters of 5×5 → 16 feature maps of 8×8. Detects combinations of edges — curves, corners, junctions.

Pool2: 2×2 max pooling → 16 maps of 4×4.

Flatten: 16 × 4 × 4 = 256 values in a 1D vector.

FC1: 256 → 120 neurons with ReLU. Learns high-level digit features.

FC2: 120 → 84 neurons with ReLU.

Output: 84 → 10 neurons with softmax. Probability for each digit 0-9.
